Connect GitHub to sync your repositories, issues, pull requests, and discussions into your Supermemory knowledge base with OAuth authentication and real-time webhooks.

## Quick Setup

### 1. Create GitHub Connection

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    import Supermemory from 'supermemory';

    const client = new Supermemory({
      apiKey: process.env.SUPERMEMORY_API_KEY!
    });

    const connection = await client.connections.create('github', {
      redirectUrl: 'https://yourapp.com/auth/github/callback',
      containerTags: ['user-123', 'github-sync'],
      documentLimit: 5000,
      metadata: {
        source: 'github',
        department: 'engineering'
      }
    });

    // Redirect user to GitHub OAuth
    window.location.href = connection.authLink;
    console.log('Auth expires in:', connection.expiresIn);
    ```
  </Tab>
  <Tab title="Python">
    ```python
    from supermemory import Supermemory
    import os

    client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

    connection = client.connections.create(
        'github',
        redirect_url='https://yourapp.com/auth/github/callback',
        container_tags=['user-123', 'github-sync'],
        document_limit=5000,
        metadata={
            'source': 'github',
            'department': 'engineering'
        }
    )

    # Redirect user to GitHub OAuth
    print(f'Redirect to: {connection.auth_link}')
    print(f'Expires in: {connection.expires_in}')
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://api.supermemory.ai/v3/connections/github" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "redirectUrl": "https://yourapp.com/auth/github/callback",
        "containerTags": ["user-123", "github-sync"],
        "documentLimit": 5000,
        "metadata": {
          "source": "github",
          "department": "engineering"
        }
      }'
    ```
  </Tab>
</Tabs>

## Supported Content Types

Supermemory syncs GitHub content automatically and converts it into searchable documents:

- **Repositories**: README files, documentation, and code comments
- **Issues**: Issue descriptions, comments, and labels  
- **Pull Requests**: PR descriptions, comments, and review discussions
- **Discussions**: GitHub Discussions posts and comments
- **Wikis**: Repository wiki pages and documentation

### Document Types

Based on the API type definitions, GitHub content is identified with these types:
- `github_repo` - Repository READMEs and documentation
- `github_issue` - Issues and their comments
- `github_pr` - Pull requests and their comments
- `github_discussion` - GitHub Discussions
- `github_wiki` - Repository wiki pages

### Conversions

To import items, supermemory converts GitHub content into markdown, and then ingests them into supermemory. This conversion is lossy, and some formatting may be lost.

## Sync Frequency

Supermemory syncs GitHub documents:
- **A document is modified or created (Webhook received)**
    - Note that not all providers are synced via webhook (Instant sync right now)
    - `GitHub` documents are synced instantaneously via webhooks
- Every **four hours** (scheduled sync)
- On **Manual Sync** (API call)
    - You can call `/v3/connections/github/sync` to sync documents manually

## What Gets Synced

### Repository Content
- **README files** - Main repository documentation
- **Wiki pages** - Repository wiki content
- **Documentation files** - Markdown files in docs/ folders
- **Code comments** - Inline code documentation

### Issues and Pull Requests
- **Issue titles and descriptions** - Main issue content
- **Issue comments** - All comments and discussions
- **PR descriptions** - Pull request details
- **PR comments** - Code review discussions
- **PR reviews** - Detailed code review feedback

### Discussions
- **General discussions** - Community discussions
- **Q&A discussions** - Question and answer threads
- **Announcements** - Project announcements

  <Tab title="Python">

    ```python
    print(f'Deleted connection: {result.id}')

    # Delete by provider and container tags
    provider_result = client.connections.delete_by_provider(
        'github',
        container_tags=['user-123']

    print(f'Deleted provider connection: {provider_result.id}')
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    # Delete by connection ID
    curl -X DELETE "https://api.supermemory.ai/v3/connections/connection_id_123" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY"

    # Response: {"id": "connection_id_123", "provider": "github"}

    # Delete by provider and container tags
    curl -X DELETE "https://api.supermemory.ai/v3/connections/github" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "containerTags": ["user-123"]
      }'

    # Response: {"id": "conn_gh123", "provider": "github"}
    ```
  </Tab>
</Tabs>

<Note>
Deleting a connection will:
- Stop all future syncs from GitHub
- Remove the OAuth authorization
- Remove webhook subscriptions
- Keep existing synced documents in Supermemory (they won't be deleted)
</Note>

## Advanced Configuration

### Bring Your Own Key (BYOK)

By default, supermemory uses its own OAuth application to connect to GitHub. However, you can configure your own OAuth application credentials for enhanced security and control.

<Danger>
  GitHub requires extensive verification and approval before you can use custom keys for production applications.
</Danger>

#### Setting up Custom GitHub OAuth Credentials

1. Set up your OAuth application on GitHub:
   - Go to https://github.com/settings/developers
   - Click "New OAuth App"
   - Fill in application details

2. Configure the redirect URL:
   ```
   https://api.supermemory.ai/v3/connections/auth/callback/github
   ```

3. Configure the client ID and client secret in the `PATCH /v3/settings` endpoint:

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    import Supermemory from 'supermemory';

    const client = new Supermemory({
      apiKey: process.env['SUPERMEMORY_API_KEY'],
    });

    // Configure GitHub custom OAuth credentials
    const settings = await client.settings.update({
      githubCustomKeyEnabled: true,
      githubClientId: "your-github-client-id",
      githubClientSecret: "your-github-client-secret"
    });
    ```
  </Tab>
  <Tab title="Python">
    ```python
    from supermemory import Supermemory

    client = Supermemory(api_key=os.environ.get("SUPERMEMORY_API_KEY"))

    # Configure GitHub custom OAuth credentials
    settings = client.settings.update(
        github_custom_key_enabled=True,
        github_client_id="your-github-client-id",
        github_client_secret="your-github-client-secret"
    )
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X PATCH "https://api.supermemory.ai/v3/settings" \
      -H "Authorization: Bearer $SUPERMEMORY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "githubCustomKeyEnabled": true,
        "githubClientId": "your-github-client-id",
        "githubClientSecret": "your-github-client-secret"
      }'
    ```
  </Tab>
</Tabs>

<Warning>
  Once you enable custom keys for GitHub, all new connections for GitHub will use your custom OAuth application. Existing connections WILL need to be re-authorized.
</Warning>

### Repository Selection

## Real-time Sync

GitHub connector uses webhooks for real-time synchronization:

- **Repository events** - New commits, pushes, releases
- **Issue events** - New issues, comments, state changes
- **PR events** - New pull requests, reviews, merges
- **Discussion events** - New discussions, comments, answers

Webhooks are automatically configured when you create a connection and cleaned up when you delete it.

<Warning>
**Important Notes:**
- OAuth tokens may expire - check `expiresAt` field
- Document processing happens asynchronously
- Use container tags consistently for filtering
- Monitor document status for failed syncs
- Webhook events are processed in real-time
- Large repositories may take time for initial sync
</Warning>
